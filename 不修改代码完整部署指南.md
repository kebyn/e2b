## 数据存储层组件清单

### 必需组件

| 组件 | 版本推荐 | 最低配置 | 用途 | 降级方案 |
|------|---------|---------|------|---------|
| PostgreSQL | 17.x | 4GB RAM, 100GB SSD | 核心数据存储 | 无 |
| Redis | 7.x | 2GB RAM | 状态共享、路由表 | 内存降级（单节点） |

### 可选但推荐组件

| 组件 | 版本推荐 | 用途 | 降级方案 | 说明 |
|------|---------|------|---------|------|
| ClickHouse | 25.x | 指标存储 | Noop 降级 | 推荐生产部署 |
| Loki | 3.x | 日志聚合 | 禁用日志收集 | API 可运行但无法收集日志 |

### 可选可省略组件

| 组件 | 用途 | 替代方案 |
|------|------|---------|
| Consul | IP 槽位分布式锁 | 本地存储（USE_LOCAL_NAMESPACE_STORAGE=true） |
| LaunchDarkly | 功能开关 | 代码内置默认值 |
| Supabase | 用户认证 | API Key 认证 |

## 1. 数据层

### 1.1 PostgreSQL（必需）

**版本推荐**: PostgreSQL 17.x（最新稳定版）
**硬件要求**: 最低 4GB RAM + 100GB SSD，生产建议 8GB+ RAM + 200GB+ SSD

#### 原生部署（高可用 - 主从复制）

```bash
# === 主节点配置 ===
# 文件: /etc/postgresql/17/main/postgresql.conf

listen_addresses = '*'
port = 5432
max_connections = 200
wal_level = replica
max_wal_senders = 10
wal_keep_size = 1GB
shared_buffers = 4GB
effective_cache_size = 12GB

# 创建复制用户
psql -U postgres -c "CREATE ROLE replicator WITH REPLICATION LOGIN PASSWORD 'password';"

# 创建应用数据库和用户
psql -U postgres -c "CREATE DATABASE e2b;"
psql -U postgres -c "CREATE USER e2b WITH PASSWORD 'password';"
psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE e2b TO e2b;"

# === 从节点配置 ===
# 初始化复制（在从节点执行）
pg_basebackup -h postgres-master -D /var/lib/postgresql/17/main \
  -U replicator -P -v -R -X stream

# 从节点 postgresql.conf（pg_basebackup 自动生成）
# primary_conninfo = 'host=postgres-master port=5432 user=replicator password=password'

# === 启动服务 ===
systemctl enable --now postgresql

# === 验证复制状态 ===
psql -U postgres -c "SELECT * FROM pg_stat_replication;"
```

#### 原生部署（单节点快速测试）

```bash
# 安装 PostgreSQL
apt install -y postgresql postgresql-contrib

# 创建数据库和用户
sudo -u postgres psql -c "CREATE DATABASE e2b;"
sudo -u postgres psql -c "CREATE USER e2b WITH PASSWORD 'password';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE e2b TO e2b;"

# 启动服务
systemctl enable --now postgresql
```

#### Docker 部署（高可用 - 主从复制）

```yaml
# docker-compose.postgres-ha.yaml
version: '3.8'

services:
  postgres-master:
    image: postgres:17-alpine
    container_name: postgres-master
    environment:
      POSTGRES_USER: e2b
      POSTGRES_PASSWORD: password
      POSTGRES_DB: e2b
      POSTGRES_REPLICATION_USER: replicator
      POSTGRES_REPLICATION_PASSWORD: replication_password
    command: >
      postgres
      -c wal_level=replica
      -c max_wal_senders=10
      -c wal_keep_size=1GB
      -c max_connections=200
      -c shared_buffers=4GB
    volumes:
      - postgres-master-data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U e2b"]
      interval: 10s
      timeout: 5s
      retries: 5

  postgres-replica:
    image: postgres:17-alpine
    container_name: postgres-replica
    environment:
      POSTGRES_USER: e2b
      POSTGRES_PASSWORD: password
      PGUSER: replicator
      PGPASSWORD: replication_password
    command: >
      postgres
      -c primary_conninfo='host=postgres-master port=5432 user=replicator password=replication_password'
      -c standby_mode=on
    volumes:
      - postgres-replica-data:/var/lib/postgresql/data
    depends_on:
      postgres-master:
        condition: service_healthy
    ports:
      - "5433:5432"

volumes:
  postgres-master-data:
  postgres-replica-data:
```

```bash
# 启动高可用 PostgreSQL
docker-compose -f docker-compose.postgres-ha.yaml up -d

# 验证复制状态
docker exec postgres-master psql -U e2b -c "SELECT * FROM pg_stat_replication;"
```

#### Docker 部署（单节点快速测试）

```bash
# 快速启动 PostgreSQL
docker run -d --name postgres \
  -e POSTGRES_USER=e2b \
  -e POSTGRES_PASSWORD=password \
  -e POSTGRES_DB=e2b \
  -p 5432:5432 \
  -v postgres-data:/var/lib/postgresql/data \
  postgres:17-alpine

# 健康检查
docker exec postgres pg_isready -U e2b
```

```bash

# 连接池配置（多实例部署）

# 认证数据库（可选独立配置）

```

#### 数据库迁移

```bash

```

### 1.2 Redis（必需）

**版本推荐**: Redis 7.x
**硬件要求**: 最低 2GB RAM，生产建议 4GB+ RAM

#### 原生部署（高可用 - Sentinel 模式）

```bash
# === 主节点配置 ===
# 文件: /etc/redis/redis.conf

bind 0.0.0.0
port 6379
appendonly yes
appendfsync everysec
maxmemory 2gb
maxmemory-policy allkeys-lru
save 900 1
save 300 10

# === 从节点配置 ===
# 文件: /etc/redis/redis.conf（在从节点上）

bind 0.0.0.0
port 6379
appendonly yes
replicaof redis-master 6379
replica-read-only yes

# === Sentinel 配置（3节点集群）===
# 文件: /etc/redis/sentinel.conf

port 26379
sentinel monitor mymaster redis-master 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 10000
sentinel parallel-syncs mymaster 1

# === 启动服务 ===
# 主节点和从节点
redis-server /etc/redis/redis.conf

# Sentinel 节点（3个）
redis-sentinel /etc/redis/sentinel.conf

# === 验证集群状态 ===
redis-cli -h redis-master INFO replication
redis-cli -h sentinel-1 -p 26379 SENTINEL master mymaster
```

#### 原生部署（单节点快速测试）

```bash
# 安装 Redis
apt install -y redis-server

# 配置（/etc/redis/redis.conf）
bind 0.0.0.0
port 6379
appendonly yes
maxmemory 1gb

# 启动服务
systemctl enable --now redis-server
```

#### Docker 部署（高可用 - Sentinel 模式）

```yaml
# docker-compose.redis-sentinel.yaml
version: '3.8'

services:
  redis-master:
    image: redis:7-alpine
    container_name: redis-master
    command: redis-server --appendonly yes --maxmemory 2gb --maxmemory-policy allkeys-lru
    volumes:
      - redis-master-data:/data
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis-replica:
    image: redis:7-alpine
    container_name: redis-replica
    command: redis-server --appendonly yes --replicaof redis-master 6379
    volumes:
      - redis-replica-data:/data
    depends_on:
      redis-master:
        condition: service_healthy
    ports:
      - "6380:6379"

  sentinel-1:
    image: redis:7-alpine
    container_name: sentinel-1
    command: redis-sentinel /etc/redis/sentinel.conf
    volumes:
      - ./sentinel.conf:/etc/redis/sentinel.conf
    depends_on:
      - redis-master
    ports:
      - "26379:26379"

  sentinel-2:
    image: redis:7-alpine
    container_name: sentinel-2
    command: redis-sentinel /etc/redis/sentinel.conf
    volumes:
      - ./sentinel.conf:/etc/redis/sentinel.conf
    depends_on:
      - redis-master
    ports:
      - "26380:26379"

  sentinel-3:
    image: redis:7-alpine
    container_name: sentinel-3
    command: redis-sentinel /etc/redis/sentinel.conf
    volumes:
      - ./sentinel.conf:/etc/redis/sentinel.conf
    depends_on:
      - redis-master
    ports:
      - "26381:26379"

volumes:
  redis-master-data:
  redis-replica-data:
```

```bash
# 创建 Sentinel 配置文件
cat > sentinel.conf << 'EOF'
port 26379
sentinel monitor mymaster redis-master 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 10000
sentinel parallel-syncs mymaster 1
EOF

# 启动 Redis Sentinel 集群
docker-compose -f docker-compose.redis-sentinel.yaml up -d

# 验证状态
docker exec sentinel-1 redis-cli -p 26379 SENTINEL master mymaster
```

#### Docker 部署（Cluster 模式 - 高级可选）

```yaml
# docker-compose.redis-cluster.yaml（6节点集群）
version: '3.8'
services:
  redis-node-1:
    image: redis:7-alpine
    command: redis-server --cluster-enabled yes --cluster-config-file nodes.conf --cluster-node-timeout 5000 --appendonly yes
    ports:
      - "7000:6379"
  redis-node-2:
    image: redis:7-alpine
    command: redis-server --cluster-enabled yes --cluster-config-file nodes.conf --cluster-node-timeout 5000 --appendonly yes
    ports:
      - "7001:6379"
  # ... 其他节点类似配置
```

```bash
# 初始化集群（所有节点启动后执行）
docker exec redis-node-1 redis-cli --cluster create \
  redis-node-1:6379 redis-node-2:6379 redis-node-3:6379 \
  redis-node-4:6379 redis-node-5:6379 redis-node-6:6379 \
  --cluster-replicas 1
```

#### 环境变量配置

```bash
# 单节点连接（格式：host:port，不支持 redis:// 前缀）
export REDIS_URL="redis:6379"

# ⚠️ 重要说明：源码不支持 Redis Sentinel 模式
# go-redis 使用 redis.NewClient，无 redis.NewFailoverClient
# 如需 Sentinel，请通过 HAProxy 暴露单一端点，如：
#   sentinel-1:26379 → HAProxy → 127.0.0.1:6379
# 然后设置 REDIS_URL="127.0.0.1:6379"

# Cluster 连接（逗号分隔多节点，不支持 redis:// 前缀）
export REDIS_CLUSTER_URL="redis-node-1:6379,redis-node-2:6379"

# TLS 配置（可选，仅集群模式生效）
export REDIS_TLS_CA_BASE64="<base64-encoded-ca>"

# 连接池配置
# API: 默认 160，多实例部署保持此值
# Client Proxy: 默认 40
# Orchestrator: 默认 5-10
export REDIS_POOL_SIZE="160"
export REDIS_MIN_IDLE_CONNS="2"
```

### 1.3 ClickHouse（可选但推荐 - 指标存储）

**版本推荐**: ClickHouse 25.x
**硬件要求**: 最低 4GB RAM + 50GB SSD，生产建议 8GB+ RAM + 200GB+ SSD
**说明**: 不配置时 API 使用 Noop 降级，不影响核心功能，但无法存储性能指标和审计日志。

#### 原生部署（高可用 - 集群模式）

```bash
# === 安装 ClickHouse ===
# Debian/Ubuntu
apt-get install -y apt-transport-https ca-certificates curl gnupg
curl -fsSL 'https://packages.clickhouse.com/rpm/lts/repodata/repomd.xml.key' | gpg --dearmor -o /usr/share/keyrings/clickhouse-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/clickhouse-keyring.gpg] https://packages.clickhouse.com/deb lts main" | tee /etc/apt/sources.list.d/clickhouse.list
apt-get update
apt-get install -y clickhouse-server clickhouse-client

# === 配置文件 ===
# 文件: /etc/clickhouse-server/config.xml

<clickhouse>
    <listen_host>0.0.0.0</listen_host>
    <http_port>8123</http_port>
    <tcp_port>9000</tcp_port>
    <interserver_http_port>9009</interserver_http_port>
    
    <max_connections>1000</max_connections>
    <keep_alive_timeout>3</keep_alive_timeout>
    
    <mark_cache_size>5368709120</mark_cache_size>  <!-- 5GB -->
    
    <path>/var/lib/clickhouse/</path>
    <tmp_path>/var/lib/clickhouse/tmp/</tmp_path>
    <user_files_path>/var/lib/clickhouse/user_files/</user_files_path>
    
    <logger>
        <level>information</level>
        <log>/var/log/clickhouse-server/clickhouse-server.log</log>
    </logger>
</clickhouse>

# === 集群配置（可选）===
# 文件: /etc/clickhouse-server/config.d/cluster.xml

<clickhouse>
    <remote_servers>
        <e2b_cluster>
            <shard>
                <replica>
                    <host>clickhouse-1</host>
                    <port>9000</port>
                </replica>
                <replica>
                    <host>clickhouse-2</host>
                    <port>9000</port>
                </replica>
            </shard>
        </e2b_cluster>
    </remote_servers>
</clickhouse>

# === 启动服务 ===
systemctl enable --now clickhouse-server

# === 验证服务 ===
clickhouse-client --query "SELECT 1"
clickhouse-client --query "SELECT * FROM system.clusters"
```

#### 原生部署（单节点快速测试）

```bash

# 启动

# 验证
```

```bash

        # ARTIFACTS_REGISTRY_PROVIDER = "AWS_ECR"
```

## 5. DNS 配置

### 5.1 沙箱 URL 构造原理

E2B 沙箱使用基于域名的路由机制，每个沙箱通过唯一的 URL 访问：

```
https://{port}-{sandbox_id}.{sandbox_domain}
```

| 参数 | 来源 | 示例 |
|------|------|------|
| `port` | 沙箱内服务端口 | `3000` |
| `sandbox_id` | API 创建沙箱时返回 | `abc123` |
| `sandbox_domain` | 服务端 `DOMAIN_NAME` 配置 | `your-domain.com` |

**完整 URL 示例：**
```
https://3000-abc123.your-domain.com
https://49983-xyz789.your-domain.com
```

### 5.2 DNS 架构流程

```
客户端请求: https://3000-abc123.your-domain.com
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  DNS: *.your-domain.com → Load Balancer IP             │
└─────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  Load Balancer (端口 443/80)                            │
│  TLS 终止 → 转发到 client-proxy:3002                    │
└─────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  client-proxy (端口 3002)                               │
│  解析 Host: "3000-abc123"                               │
│    - port = 3000                                        │
│    - sandbox_id = abc123                                │
│  查询 Redis: sandbox_id → nodeIP                        │
│  转发到 nodeIP:5007                                     │
└─────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────┐
│  orchestrator proxy (端口 5007)                         │
│  转发到 sandbox envd (端口 49983)                       │
└─────────────────────────────────────────────────────────┘
```

### 5.3 必需的 DNS 记录

| 记录类型 | 名称 | 值 | 用途 |
|----------|------|-----|------|
| A | `api.your-domain.com` | API 服务 IP | API 端点 |
| A (通配符) | `*.your-domain.com` | LB/Ingress IP | 所有沙箱请求 |

#### BIND Zone 示例

```dns
$ORIGIN your-domain.com.

; API 端点
api              IN  A     10.0.0.100

; 沙箱通配符域名
*                IN  CNAME sandbox-lb
sandbox-lb       IN  A     10.0.0.200
```

#### Kubernetes Ingress 示例

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: sandbox-wildcard
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
spec:
  tls:
    - hosts:
        - "*.your-domain.com"
        - "api.your-domain.com"
      secretName: wildcard-tls
  rules:
    - host: "*.your-domain.com"
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: client-proxy
                port:
                  number: 3002
    - host: "api.your-domain.com"
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: api
                port:
                  number: 80
```

### 5.4 端口映射参考

| 组件 | 端口 | 环境变量 | 说明 |
|------|------|----------|------|
| Load Balancer | 443/80 | - | 外部入口 |
| client-proxy | 3002 | `PROXY_PORT` | 接收外部沙箱请求 |
| orchestrator proxy | 5007 | `PROXY_PORT` | 内部转发 |
| orchestrator gRPC | 5008 | `GRPC_PORT` | 服务间通信 |
| envd | 49983 | - | 沙箱内部服务 |

### 5.5 TLS 证书配置

**证书要求：**
- 必须覆盖 `*.your-domain.com`（通配符证书）
- 建议同时覆盖 `api.your-domain.com`

**Let's Encrypt 示例：**

```bash
# 使用 certbot 申请通配符证书
certbot certonly --manual --preferred-challenges dns \
  -d "*.your-domain.com" \
  -d "your-domain.com" \
  --email admin@your-domain.com
```

**证书配置（Nginx）：**

```nginx
server {
    listen 443 ssl;
    server_name *.your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    location / {
        proxy_pass http://e2b_proxy;
        proxy_set_header Host $host;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
    }
}
```

### 5.6 客户端配置

| 场景 | 配置 | 说明 |
|------|------|------|
| 标准私有化 | `E2B_API_KEY` + `E2B_DOMAIN=your-domain.com` | 通配符 DNS 已配置 |
| 本地开发 | `E2B_API_KEY` + `E2B_DEBUG=true` | 直接连 localhost |
| 自定义代理 | `E2B_API_KEY` + `E2B_SANDBOX_URL=https://proxy.example.com` | 绕过 DNS 构造 |

### 5.7 DNS 故障排查

| 问题 | 诊断命令 | 解决方案 |
|------|----------|----------|
| DNS 解析失败 | `nslookup *.your-domain.com` | 添加通配符 A 记录 |
| TLS 证书错误 | `openssl s_client -connect abc123.your-domain.com:443` | 确认证书覆盖通配符域名 |
| 连接超时 | `curl -v https://3000-abc123.your-domain.com` | 检查 client-proxy 服务状态 |
| 404 Not Found | 检查 sandbox_id | 沙箱可能已过期或不存在 |
| 端口无法访问 | `telnet your-domain.com 443` | 检查防火墙规则 |
