## 组件清单

| 组件 | 必需 | 用途 | 部署方式 |
|------|------|------|----------|
| PostgreSQL | ✅ 必需 | 核心数据存储 | 主从复制 |

| Nomad | ✅ 必需 | 服务发现、任务编排 | 3 节点集群 |
| API | ✅ 必需 | HTTP/gRPC 接口 | 3 实例 |

| Orchestrator | ✅ 必需 | Sandbox 运行管理 | Nomad system job |
| Template Manager | ⚠️ 按需 | 模板构建 | Nomad service job |
| Dashboard API | ⚠️ 可选 | Web 管理后端 | 2 实例 |
| Dashboard Frontend | ⚠️ 可选 | Web 管理前端 | 2 实例 |
| ClickHouse | ⚠️ 可选 | 指标存储 | 单节点 |
| Loki | ⚠️ 可选 | 日志聚合 | 单节点 |
| Consul | ⚠️ 可选 | IP 槽位分布式锁 | 可用本地存储替代 |
| LaunchDarkly | ⚠️ 可选 | 功能开关 | 不配置即用默认值 |
| Supabase | ⚠️ 可选 | 用户认证 | 不配置即用 API Key |

## 1. 数据层

### 1.1 PostgreSQL（必需）

```bash
# 文件: /etc/postgresql/17/main/postgresql.conf

listen_addresses = '*'
port = 5432
max_connections = 200
wal_level = replica
max_wal_senders = 10
wal_keep_size = 1GB
shared_buffers = 4GB
effective_cache_size = 12GB

# 主节点: 创建复制用户
psql -U postgres -c "CREATE ROLE replicator WITH REPLICATION LOGIN PASSWORD 'password';"

# 从节点: 初始化复制
pg_basebackup -h postgres-master -D /var/lib/postgresql/17/main \
  -U replicator -P -v -R -X stream

# 运行数据库迁移
export POSTGRES_CONNECTION_STRING="postgresql://e2b:password@postgres:5432/e2b?sslmode=disable"
make -C packages/db migrate
```

### 1.2 Redis（必需）

```bash
# 文件: /etc/redis/redis.conf
bind 0.0.0.0
port 6379
appendonly yes
maxmemory 2gb
maxmemory-policy allkeys-lru

# 从节点添加:
replicaof redis-master 6379

# Sentinel 配置 (/etc/redis/sentinel.conf)
port 26379
sentinel monitor mymaster redis-master 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel failover-timeout mymaster 10000

# 启动
redis-server /etc/redis/redis.conf        # 数据节点
redis-sentinel /etc/redis/sentinel.conf   # Sentinel 节点
```

### 1.3 ClickHouse（可选，指标存储）

## 2. 服务发现层

### 2.1 Nomad 集群（必需）

```bash
# 文件: /etc/nomad.d/server.hcl

datacenter = "dc1"
data_dir   = "/opt/nomad/data"

server {
  enabled          = true
  bootstrap_expect = 3
  server_join {
    retry_join = ["nomad-1:4648", "nomad-2:4648", "nomad-3:4648"]
  }
}

acl { enabled = true }
```

```bash
# 计算节点配置: /etc/nomad.d/client.hcl

datacenter = "dc1"
data_dir   = "/opt/nomad/data"

client {
  enabled = true
  server_join {
    retry_join = ["nomad-1:4648", "nomad-2:4648", "nomad-3:4648"]
  }
  reserved {
    cpu    = 500
    memory = 1024
  }
}

acl { enabled = true }
```

```bash
# 启动
nomad agent -config=/etc/nomad.d/server.hcl  # 服务器节点
nomad agent -config=/etc/nomad.d/client.hcl  # 计算节点

# 初始化 ACL
nomad acl bootstrap
export NOMAD_TOKEN="<bootstrap-secret-id>"

# 创建策略和 Token
nomad acl policy apply e2b - <<EOF
namespace "default" { policy = "write" }
node { policy = "read" }
EOF

nomad acl token create -name="e2b-api" -policy="e2b"
```

## 3. 应用层

### 3.1 API（必需，3 实例）

```bash
# 文件: /usr/local/bin/start-api.sh
#!/bin/bash
export NODE_ID="api-$(hostname)"
export ENVIRONMENT="prod"

# === 数据库连接（必需） ===
export POSTGRES_CONNECTION_STRING="postgresql://e2b:pass@postgres:5432/e2b"
export DB_MAX_OPEN_CONNECTIONS="40"
export DB_MIN_IDLE_CONNECTIONS="5"
export AUTH_DB_MAX_OPEN_CONNECTIONS="20"
export AUTH_DB_MIN_IDLE_CONNECTIONS="5"

# === Redis 连接（多实例必需） ===
export REDIS_URL="redis://sentinel-1:26379/0?sentinel=mymaster"
export REDIS_POOL_SIZE="160"
# 可选：Redis Cluster 或 TLS
# export REDIS_CLUSTER_URL="redis://cluster:6379"
# export REDIS_TLS_CA_BASE64="<base64-ca>"

# === Nomad 连接（必需） ===
export NOMAD_ADDRESS="http://nomad:4646"
export NOMAD_TOKEN="<acl-token>"

# === Sandbox 存储配置（多实例必需设为 redis） ===
export SANDBOX_STORAGE_BACKEND="redis"  # 多实例必须为 redis，单实例可用 memory

# === 模板存储配置 ===
export TEMPLATE_BUCKET_NAME="e2b-templates"
export BUILD_CACHE_BUCKET_NAME="e2b-build-cache"
export STORAGE_PROVIDER="GCPBucket"  # 可选: GCPBucket, AWSBucket, Local
export GOOGLE_SERVICE_ACCOUNT_BASE64="<base64-sa>"

# === 日志后端（必需） ===
export LOKI_URL="http://loki:3100"
export LOKI_USER=""    # 可选，如有认证
export LOKI_PASSWORD="" # 可选，如有认证

# === Volume Token 配置（必需） ===
# 用于签名 volume 访问令牌
export VOLUME_TOKEN_ISSUER="local.e2b.dev"
export VOLUME_TOKEN_SIGNING_METHOD="ES256"  # 支持: ES256, RS256, HS256, EdDSA
export VOLUME_TOKEN_SIGNING_KEY="ECDSA:<base64-encoded-pem>"
export VOLUME_TOKEN_SIGNING_KEY_NAME="prod-2025-01"
# export VOLUME_TOKEN_DURATION="1h"  # 可选，默认 1h

# === 可选配置 ===
export ADMIN_TOKEN="<admin-token>"
export API_GRPC_PORT="5009"
export DOMAIN_NAME="e2b.example.com"

# === 可选：监控和分析 ===
# export CLICKHOUSE_CONNECTION_STRING="clickhouse://..."
# export POSTHOG_API_KEY=""

# === 可选：Supabase 认证（Dashboard 必需） ===
# export SUPABASE_JWT_SECRETS="secret1,secret2"

exec /usr/local/bin/api --port 3000
```

> **重要**: `VOLUME_TOKEN_*` 系列变量是必需的，否则 API 无法启动。签名密钥需要使用 PEM 格式的私钥并 Base64 编码。

```bash
# 启动 3 个实例
NODE_ID=api-1 ./start-api.sh  # 端口 3000
NODE_ID=api-2 ./start-api.sh  # 端口 3000
NODE_ID=api-3 ./start-api.sh  # 端口 3000
```

### 3.3 Orchestrator + Template Manager（通过 Nomad 部署）

**说明**: Orchestrator 和 Template Manager 使用同一个二进制 `orchestrator`，通过 `ORCHESTRATOR_SERVICES` 环境变量区分角色。

#### 前置条件（每计算节点）

```bash
# 加载内核模块
sudo modprobe nbd nbds_max=256    # 或更高值（代码默认64，生产建议256+）
sudo modprobe tun
sudo sysctl -w vm.nr_hugepages=2048
echo "vm.nr_hugepages=2048" | sudo tee -a /etc/sysctl.conf
echo "vm.unprivileged_userfaultfd=1" | sudo tee -a /etc/sysctl.conf

# 创建目录（注意：代码默认路径为 /fc-*，可通过环境变量覆盖）
sudo mkdir -p /opt/e2b/{orchestrator,fc-vm,kernels,firecracker,envd,busybox}

# 下载内核和 Firecracker
make download-public-kernels download-public-firecrackers

#将下载的文件复制到目标目录
cp -r packages/fc-kernels/* /opt/e2b/kernels/
cp -r packages/fc-versions/builds/* /opt/e2b/firecracker/
```

> **路径说明**: 代码默认路径为`/fc-kernels`, `/fc-versions`, `/fc-envd`, `/fc-busybox`， `/fc-vm`， `/orchestrator`。
> 可通过设置环境变量覆盖为自定义路径。

#### Nomad 任务定义

```bash
# 文件: e2b-orchestrator.nomad
# 用途: Orchestrator (Sandbox 运行管理，必需)

job "e2b-orchestrator" {
  datacenters = ["dc1"]
  type        = "system"  # 每个计算节点自动运行一个

  group "orchestrator" {
    network {
      port "grpc"  { static = 5008 }
      port "proxy" { static = 5007 }
    }

    task "orchestrator" {
      driver = "exec"
      config { command = "/usr/local/bin/orchestrator" }

      env {
        # === 节点标识 ===
        NODE_ID                     = "${node.unique.name}"
        NODE_IP                     = "${attr.unique.network.ip-address}"
        NODE_LABELS                 = "${meta.node_labels}"
        ENVIRONMENT                 = "prod"

        # === 服务选择 ===
        ORCHESTRATOR_SERVICES       = "orchestrator"              # 关键: 仅 Orchestrator

        # === Redis连接 ===
        REDIS_URL                   = "redis://sentinel-1:26379/0?sentinel=mymaster"
        REDIS_POOL_SIZE             = "20"
        REDIS_MIN_IDLE_CONNS        = "2"
        # REDIS_CLUSTER_URL         = ""
        # REDIS_TLS_CA_BASE64       = ""

        # === 存储配置 ===
        TEMPLATE_BUCKET_NAME        = "e2b-templates"
        BUILD_CACHE_BUCKET_NAME     = "e2b-build-cache"
        STORAGE_PROVIDER            = "GCPBucket"  # 或 AWSBucket, Local
        PROVIDER                    = "gcp"        # 或 aws

        # === 本地存储（避免 Consul 依赖） ===
        USE_LOCAL_NAMESPACE_STORAGE = "true"

        # === 路径配置（必须与实际目录一致） ===
        ORCHESTRATOR_BASE_PATH      = "/opt/e2b/orchestrator"
        SANDBOX_DIR                 = "/opt/e2b/fc-vm"
        HOST_KERNELS_DIR            = "/opt/e2b/kernels"
        FIRECRACKER_VERSIONS_DIR    = "/opt/e2b/firecracker"
        HOST_ENVD_PATH              = "/opt/e2b/envd/envd"
        HOST_BUSYBOX_DIR            = "/opt/e2b/busybox"         # 不可遗漏！
        SHARED_CHUNK_CACHE_PATH     = "/opt/e2b/orchestrator/cache"

        # === 端口配置 ===
        GRPC_PORT                   = "5008"
        PROXY_PORT                  = "5007"

        # === 可选：监控 ===
        # CLICKHOUSE_CONNECTION_STRING = "clickhouse://..."
        # LAUNCH_DARKLY_API_KEY       = ""

        # === 云平台特定配置 ===
        # GCP:
        # ARTIFACTS_REGISTRY_PROVIDER = "GCP_ARTIFACTS"
        # AWS:
        # ARTIFACTS_REGISTRY_PROVIDER = "AWS_ECR"
        # AWS_REGION                   = "us-west-2"

        # === 可选：持久卷挂载 ===
        # PERSISTENT_VOLUME_MOUNTS    = "name1=/path1,name2=/path2"
      }

      resources {
        cpu    = 4000
        memory = 8192
      }
    }
  }
}

# 提交
nomad job run -token="$NOMAD_TOKEN" e2b-orchestrator.nomad
```

## 4. 负载均衡

```nginx
# /etc/nginx/conf.d/e2b.conf

upstream e2b_api {
    least_conn;
    server api-1:3000 max_fails=3 fail_timeout=30s;
    server api-2:3000 max_fails=3 fail_timeout=30s;
    server api-3:3000 max_fails=3 fail_timeout=30s;
}

upstream e2b_proxy {
    least_conn;
    server proxy-1:3002 max_fails=3 fail_timeout=30s;
    server proxy-2:3002 max_fails=3 fail_timeout=30s;
}

# Dashboard (可选)
upstream e2b_dashboard_api {
    least_conn;
    server dashboard-api-1:3010 max_fails=3 fail_timeout=30s;
    server dashboard-api-2:3010 max_fails=3 fail_timeout=30s;
}

upstream e2b_dashboard {
    least_conn;
    server dashboard-1:3000 max_fails=3 fail_timeout=30s;
    server dashboard-2:3000 max_fails=3 fail_timeout=30s;
}

server {
    listen 80;
    server_name api.e2b.example.com;
    location / {
        proxy_pass http://e2b_api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 300s;
    }
}

server {
    listen 80;
    server_name *.sandbox.e2b.example.com;
    location / {
        proxy_pass http://e2b_proxy;
        proxy_set_header Host $host;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
    }
}

# Dashboard (可选)
server {
    listen 80;
    server_name dashboard.e2b.example.com;

    location / {
        proxy_pass http://e2b_dashboard;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/ {
        proxy_pass http://e2b_dashboard_api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

```bash
# 启动 (PM2 示例)
pm2 start npm --name "e2b-dashboard" -- start

# 或 Docker
docker run -d --name e2b-dashboard \
  -p 3000:3000 \
  -e NEXT_PUBLIC_API_URL=http://dashboard-api:3010 \
  -e NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co \
  -e NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key \
  e2b/dashboard:latest
```

### 5.4 Loki（可选，日志聚合）

```bash
docker run -d --name loki -p 3100:3100 \
  -v /var/lib/loki:/loki grafana/loki:latest

export LOKI_URL="http://loki:3100"
```

### 5.5 Consul（可选，IP 槽位分布式锁）

```bash
# 默认使用本地存储，无需 Consul
export USE_LOCAL_NAMESPACE_STORAGE="true"

# 如需使用 Consul:
export CONSUL_TOKEN="<token>"
# 并删除 USE_LOCAL_NAMESPACE_STORAGE 配置
```

## 6. 部署顺序

```bash
# 1. PostgreSQL 主从 + 数据库迁移
# 2. Redis Sentinel
# 3. Nomad 3 节点集群 + ACL 初始化
# 4. API 3 实例
# 5. Client Proxy 2+ 实例
# 6. Orchestrator (Nomad system job)
# 7. Template Manager (Nomad service job，按需)
# 8. 负载均衡 (Nginx/HAProxy)
# 9. Dashboard API 2 实例 (可选，需先配置 Supabase)
# 10. Dashboard Frontend (可选)
# 11. 其他可选组件 (ClickHouse, Loki)
```

## 7. 健康检查

```bash
# API
curl http://api-1:3000/health
curl http://api-2:3000/health
curl http://api-3:3000/health

# Client Proxy
curl http://proxy-1:3003/
curl http://proxy-2:3003/

# Orchestrator / Template Manager (通过 Nomad)
nomad status e2b-orchestrator
nomad status e2b-template-manager

# Dashboard API (可选)
curl http://dashboard-api-1:3010/health
curl http://dashboard-api-2:3010/health

# Nomad 集群
nomad server members
nomad node status

# Redis
redis-cli -h sentinel-1 -p 26379 SENTINEL master mymaster

# PostgreSQL
psql -h postgres -U e2b -c "SELECT 1;"
```
