# 不修改代码 - 高可用生产部署方案

---

## 核心结论

| 组件 | 是否必须 | 降级方案 | 影响 |
|------|----------|----------|------|
| **Nomad** | ✅ **必须** | 无降级 | 多节点 Orchestrator 发现依赖此组件 |
| **Redis** | ✅ **必须** | 内存降级 | 无 Redis 无法跨 API 实例共享 sandbox 状态 |
| **PostgreSQL** | ✅ **必须** | 无降级 | 核心数据存储 |
| **Consul** | ❌ 可选 | 本地存储 | 每节点独立 IP 池 |
| **LaunchDarkly** | ❌ 可选 | 默认值 | 功能开关不可动态调整 |
| **ClickHouse** | ❌ 可选 | Noop | 无监控指标 |
| **Supabase** | ❌ 可选 | API Key | 无用户级认证 |

## 2. 必需组件部署

### 2.1 Nomad 集群（必需）

**为什么必须：** 代码中 API 通过 Nomad 发现 Orchestrator 节点，没有降级机制。

```bash
# === Nomad 服务器节点配置 ===
# 文件: /etc/nomad.d/nomad.hcl

data_dir  = "/opt/nomad/data"
bind_addr = "0.0.0.0"

server {
  enabled          = true
  bootstrap_expect = 3
  
  # 服务发现需要
  server_join {
    retry_join = [
      "nomad-1.example.com",
      "nomad-2.example.com", 
      "nomad-3.example.com"
    ]
  }
}

client {
  enabled = true
  # 允许 Orchestrator 运行
  node_class = "compute"
  
  # 保留资源给系统
  reserved {
    cpu    = 500
    memory = 1024
  }
}

acl {
  enabled = true
}

telemetry {
  publish_allocation_metrics = true
  publish_node_metrics       = true
}
```

```bash
# === 启动 Nomad 集群 ===

# 节点 1
nomad agent -config=/etc/nomad.d/nomad.hcl

# 服务器加入（其他节点）
nomad server join nomad-1.example.com:4648

# 初始化 ACL
nomad acl bootstrap

# 创建 Orchestrator 任务策略
nomad acl policy apply orchestrator - <<EOF
namespace "default" {
  policy = "read"
  capabilities = ["submit-job", "read-job"]
}
node {
  policy = "read"
}
EOF

# 创建 Token
nomad acl token create -name="e2b-api" -policy="orchestrator"
```

```bash
# === Nomad Orchestrator 任务定义 ===
# 文件: orchestrator.nomad

job "orchestrator" {
  datacenters = ["dc1"]
  type        = "system"  # 每个节点运行一个

  group "orchestrator" {
    network {
      port "grpc" { static = 5008 }
      port "proxy" { static = 5007 }
    }

    task "orchestrator" {
      driver = "exec"

      config {
        command = "/usr/local/bin/orchestrator"
      }

      env {
        NODE_ID                        = "${node.unique.name}"
        ENVIRONMENT                    = "prod"
        GRPC_PORT                      = "5008"
        PROXY_PORT                     = "5007"
        REDIS_URL                      = "redis://redis:6379"
        TEMPLATE_BUCKET_NAME           = "e2b-templates"
        BUILD_CACHE_BUCKET_NAME        = "e2b-build-cache"
        USE_LOCAL_NAMESPACE_STORAGE    = "true"
        ORCHESTRATOR_BASE_PATH         = "/opt/e2b/orchestrator"
        HOST_KERNELS_DIR               = "/opt/e2b/kernels"
        FIRECRACKER_VERSIONS_DIR       = "/opt/e2b/firecracker"
        HOST_ENVD_PATH                 = "/opt/e2b/envd/envd"
      }

      resources {
        cpu    = 4000
        memory = 8192
      }
    }
  }
}
```

### 2.2 Redis 高可用

**为什么必须：** 多个 API 实例需要共享 sandbox 路由表和状态。

```bash
# === Redis Sentinel 配置 ===
# 文件: /etc/redis/redis.conf

bind 0.0.0.0
port 6379
daemonize no
supervised systemd

# 持久化
appendonly yes
appendfsync everysec

# 主从复制 (每个副本节点配置)
replicaof redis-master 6379

# 内存限制
maxmemory 2gb
maxmemory-policy allkeys-lru
```

```bash
# === Redis Sentinel 配置 ===
# 文件: /etc/redis/sentinel.conf

port 26379
sentinel monitor mymaster redis-master 6379 2
sentinel down-after-milliseconds mymaster 5000
sentinel parallel-syncs mymaster 1
sentinel failover-timeout mymaster 10000
```

```bash
# === 启动 Redis ===

# 主节点
redis-server /etc/redis/redis.conf

# 从节点 (配置文件中添加 replicaof)
redis-server /etc/redis/redis.conf

# Sentinel (每个节点)
redis-sentinel /etc/redis/sentinel.conf
```

### 2.3 PostgreSQL 高可用

```bash
# === 基础配置 ===
# 文件: /etc/postgresql/17/main/postgresql.conf

listen_addresses = '*'
port = 5432
max_connections = 200

# WAL 配置 (用于主从复制)
wal_level = replica
max_wal_senders = 10
wal_keep_size = 1GB

# 性能
shared_buffers = 4GB
effective_cache_size = 12GB
work_mem = 64MB
maintenance_work_mem = 512MB
```

```bash
# === 主从复制设置 ===

# 主节点: 创建复制用户
psql -U postgres -c "CREATE ROLE replicator WITH REPLICATION LOGIN PASSWORD 'password';"

# 从节点: 基础备份
pg_basebackup -h postgres-master -D /var/lib/postgresql/17/main -U replicator -P -v -R -X stream

# 从节点配置 standby.signal
touch /var/lib/postgresql/17/main/standby.signal

# 从节点连接配置
echo "primary_conninfo = 'host=postgres-master port=5432 user=replicator password=password'" >> postgresql.auto.conf
```

### 3.2 负载均衡配置

```nginx
# /etc/nginx/conf.d/e2b-api.conf

upstream e2b_api {
    least_conn;
    server api-1:3000;
    server api-2:3001;
    server api-3:3002;
}

upstream e2b_proxy {
    least_conn;
    server proxy-1:3002;
    server proxy-2:3003;
}

# API 服务
server {
    listen 80;
    server_name api.e2b.example.com;

    location / {
        proxy_pass http://e2b_api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}

# Sandbox 代理 (*.sandbox.e2b.example.com)
server {
    listen 80;
    server_name *.sandbox.e2b.example.com;

    location / {
        proxy_pass http://e2b_proxy;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
    }
}
```

## 5. Orchestrator 部署

### 5.1 每个计算节点部署

```bash
# === Orchestrator 节点配置 ===
# 节点 1: 10.0.1.10
# 节点 2: 10.0.1.11
# 节点 3: 10.0.1.12

# === 前置条件 ===

# 加载内核模块
sudo modprobe nbd nbds_max=256
sudo modprobe tun

# 设置大页内存
sudo sysctl -w vm.nr_hugepages=2048
echo "vm.nr_hugepages=2048" | sudo tee -a /etc/sysctl.conf

# 创建目录
sudo mkdir -p /opt/e2b/{orchestrator,kernels,firecracker,envd,fc-vm}
```

```bash
# === Orchestrator 启动脚本 ===
# 文件: /usr/local/bin/start-orchestrator.sh

#!/bin/bash

export NODE_ID=$(hostname)
export ENVIRONMENT="prod"
export NODE_IP=$(hostname -I | awk '{print $1}')

# 网络配置 - 使用本地存储避免 Consul 依赖
export USE_LOCAL_NAMESPACE_STORAGE="true"

# Redis
export REDIS_URL="redis://redis-sentinel:26379/0?sentinel=mymaster"

# 存储
export TEMPLATE_BUCKET_NAME="e2b-templates"
export BUILD_CACHE_BUCKET_NAME="e2b-build-cache"
export STORAGE_PROVIDER="GCPBucket"

# 路径
export ORCHESTRATOR_BASE_PATH="/opt/e2b/orchestrator"
export SANDBOX_DIR="/opt/e2b/fc-vm"
export HOST_KERNELS_DIR="/opt/e2b/kernels"
export FIRECRACKER_VERSIONS_DIR="/opt/e2b/firecracker"
export HOST_ENVD_PATH="/opt/e2b/envd/envd"

# 端口
export GRPC_PORT="5008"
export PROXY_PORT="5007"

# 可选: ClickHouse 监控
# export CLICKHOUSE_CONNECTION_STRING="clickhouse://user:pass@clickhouse:9000/default"

exec /usr/local/bin/orchestrator
```

```bash
# === Systemd 服务 ===
# 文件: /etc/systemd/system/orchestrator.service

[Unit]
Description=E2B Orchestrator
After=network.target
Wants=network-online.target

[Service]
Type=simple
User=root
ExecStart=/usr/local/bin/start-orchestrator.sh
Restart=always
RestartSec=10

# 资源限制
LimitNOFILE=65536
LimitNPROC=65536

# 环境文件
EnvironmentFile=/etc/e2b/orchestrator.env

[Install]
WantedBy=multi-user.target
```

```bash
# === 启动 Orchestrator ===

sudo systemctl daemon-reload
sudo systemctl enable orchestrator
sudo systemctl start orchestrator
```

## 6. Nomad 任务注册

```bash
# === 通过 Nomad 注册 Orchestrator 任务 ===
# 文件: orchestrator.nomad

job "e2b-orchestrator" {
  datacenters = ["dc1"]
  type        = "system"  # 每个客户端节点运行一个实例

  group "orchestrator" {
    network {
      port "grpc" { static = 5008 }

    }

    task "orchestrator" {
      driver = "exec"

      config {
        command = "/usr/local/bin/orchestrator"

      env {
        NODE_ID                        = "${node.unique.name}"
        NODE_IP                        = "${attr.unique.network.ip-address}"
        ENVIRONMENT                    = "prod"
        GRPC_PORT                      = "5008"
        PROXY_PORT                     = "5007"
        REDIS_URL                      = "redis://redis-sentinel:26379/0?sentinel=mymaster"
        TEMPLATE_BUCKET_NAME           = "e2b-templates"

        HOST_KERNELS_DIR               = "/opt/e2b/kernels"

        HOST_ENVD_PATH                 = "/opt/e2b/envd/envd"
```

## 9. 故障恢复

### 9.1 API 节点故障

```bash
# 自动: 负载均衡器检测健康检查失败后自动剔除
# 手动: 重启服务
systemctl restart e2b-api
```

### 9.2 Orchestrator 节点故障

```bash
# Nomad 自动重新调度到其他节点
# 或手动在新节点启动
nomad job restart e2b-orchestrator
```

### 9.3 Redis 故障

```bash
# Sentinel 自动故障转移
# 检查新主节点
redis-cli -h sentinel-1 -p 26379 SENTINEL get-master-addr-by-name mymaster
```

### 9.4 PostgreSQL 故障

```bash
# 手动切换到从节点
pg_ctlcluster 17 main promote

# 更新连接字符串
export POSTGRES_CONNECTION_STRING="postgresql://e2b:pass@postgres-replica:5432/e2b"
```
