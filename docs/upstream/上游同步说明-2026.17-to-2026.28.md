## 3. 配置与运行时变化

### 3.1 `.env.gcp.template`

新增或调整的关键项：

- `DOCKER_REVERSE_PROXY_VERSION`
- `CLIENT_PROXY_VERSION`
- `CLICKHOUSE_MIGRATOR_VERSION`
- `GCP_REDIS_NODE_TYPE`
- `CLIENT_PROXY_OIDC_ISSUER_URL`
- `ORY_SDK_URL`
- `ORY_ISSUER_URL`

删除的旧 dashboard 开关：

- `ENABLE_AUTH_USER_SYNC_BACKGROUND_WORKER`
- `ENABLE_BILLING_HTTP_TEAM_PROVISION_SINK`

说明变化：

- Postgres 注释从 Supabase 示例收敛为普通 Postgres connection string。
- Ory Project API token 由 cloud secret 管理，模板只留下说明，不直接要求明文写入 env。

### 3.2 `.env.aws.template`

新增关键项：

- `DOCKER_REVERSE_PROXY_VERSION`
- `CLIENT_PROXY_VERSION`
- `CLICKHOUSE_MIGRATOR_VERSION`

这些变量意味着上游支持从 E2B artifact registry 拉取已发布的预构建镜像，再镜像到自有 registry；本地“从源码构建所有服务”的文档需要明确这是可选路径。

GCP/AWS 模板还新增了 sandbox firewall 的 `ALLOW_SANDBOX_INTERNAL_CIDRS` 注释说明，但在 `2026.28` 模板中它是注释行，不是实际赋值变量。

### 3.3 API 配置

`infra/packages/api/internal/cfg/model.go` 的关键变化：

- 新增 `CLICKHOUSE_CONNECTION_STRINGS`，支持用分号分隔多个 ClickHouse 连接串。
- 新增 `SERVICE_DISCOVERY_PROVIDER`，可选：
  - `nomad`：默认，查询 Nomad HTTP API。
  - `kubernetes`：查询 K8s in-cluster API。
  - `local`：使用静态 orchestrator 地址。
- Nomad discovery 新增：
  - `NOMAD_ORCHESTRATOR_SERVICE_NAMES`
  - `NOMAD_ORCHESTRATOR_LEGACY_DISCOVERY_ENABLED`
- Local discovery 新增：
  - `LOCAL_ORCHESTRATOR_ADDRESS`
- K8s discovery 新增：
  - `K8S_NAMESPACE`
  - `K8S_ORCHESTRATOR_POD_LABEL_SELECTOR`
  - `K8S_TEMPLATE_MANAGER_POD_LABEL_SELECTOR`
- API gRPC 端口从单一 `API_GRPC_PORT` 拆成：
  - `API_INTERNAL_GRPC_PORT`，默认 `5009`
  - `API_EDGE_GRPC_PORT`，默认 `5109`
- 新增 `CLIENT_PROXY_OIDC_ISSUER_URL`。
- `SUPABASE_JWT_SECRETS` 被 `AUTH_PROVIDER_CONFIG` 取代。
- `DEFAULT_KERNEL_VERSION` 不再在 API config 中兜底。
- `SANDBOX_STORAGE_BACKEND` 从 API config 中移除。
- Volume token 新增 `VOLUME_TOKEN_ENABLED`，当设为 `false` 时可不提供 signing env。

### 3.4 Client Proxy 配置

`infra/packages/client-proxy/internal/cfg/model.go` 的关键变化：

- 旧 `API_GRPC_ADDRESS` 拆分为：
  - `API_INTERNAL_GRPC_ADDRESS`
  - `API_EDGE_GRPC_ADDRESS`
- 新增 edge gRPC OAuth：
  - `API_EDGE_GRPC_OAUTH_CLIENT_ID`
  - `API_EDGE_GRPC_OAUTH_CLIENT_SECRET`
  - `API_EDGE_GRPC_OAUTH_TOKEN_URL`

这会影响所有手工部署脚本、Nomad job、systemd unit 和 Nginx/proxy 文档中的 API/proxy 通信说明。

### 3.5 Dashboard API 配置

`infra/packages/dashboard-api/internal/cfg/model.go` 的关键变化：

- 新增 `CLICKHOUSE_CONNECTION_STRINGS`。
- `SUPABASE_JWT_SECRETS` 替换为 `AUTH_PROVIDER_CONFIG`。
- 删除 `SUPABASE_DB_CONNECTION_STRING`。
- 删除：
  - `ENABLE_AUTH_USER_SYNC_BACKGROUND_WORKER`
  - `ENABLE_BILLING_HTTP_TEAM_PROVISION_SINK`
- 新增 Ory 配置：
  - `ORY_SDK_URL`
  - `ORY_PROJECT_API_TOKEN`
  - `ORY_ISSUER_URL`
- 新增 `DOMAIN_NAME`。
- 启动校验要求：
  - 必须配置 `REDIS_URL` 或 `REDIS_CLUSTER_URL`。
  - 必须配置 Ory SDK URL、Project API Token、Issuer URL。
  - 当 `AUTH_PROVIDER_CONFIG.jwt[]` 存在时，`ORY_ISSUER_URL` 必须匹配其中一个 issuer。

## 5. 服务发现与部署拓扑变化

`2026.28` 引入更明确的服务发现抽象：

| Provider | 变量 | 适用场景 |
|----------|------|----------|
| `nomad` | `SERVICE_DISCOVERY_PROVIDER=nomad` | 官方 Nomad 部署路径，默认值 |
| `kubernetes` | `SERVICE_DISCOVERY_PROVIDER=kubernetes` | K8s 部署路径，通过 pod label selector 查找服务 |
| `local` | `SERVICE_DISCOVERY_PROVIDER=local` | 本地开发或静态单 orchestrator |

这会影响：

- `K8s私有化部署指南.md` 已使用上游 Kubernetes discovery，不再把 K8s 描述为必须自行改代码适配。
- `不修改代码完整部署指南.md` 已补充 Nomad service names 和 legacy discovery 开关。
- `私有化部署组件分析.md` 已将 K8s 和 local 作为当前源码支持项重新评估。
- `启动参数详解.md` 已记录 discovery 相关变量和默认值。
