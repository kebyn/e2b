## 1. API (`infra/packages/api/bin/api`)

### 命令行参数

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `-port` | `int` | ❌ | `80` | HTTP 服务监听端口 |

**示例：**
```bash
./infra/packages/api/bin/api -port 3000
```

## 2. Orchestrator (`infra/packages/orchestrator/bin/orchestrator`)

### 命令行参数
无命令行参数，全部通过环境变量配置。

**示例：**
```bash
NODE_ID=node-1 TEMPLATE_BUCKET_NAME=my-bucket ./infra/packages/orchestrator/bin/orchestrator
```

### 环境变量

#### 基础环境

| 变量 | 类型 | 必需 | 默认值 | 枚举值 | 说明 |
|------|------|------|--------|--------|------|
| `NODE_ID` | `string` | ✅ | - | - | 节点唯一标识 |
| `ENVIRONMENT` | `string` | ❌ | `"prod"` | `prod` `dev` `local` | 运行环境 |
| `E2B_DEBUG` | `string` | ❌ | `"false"` | `true` `false` | 启用 debug 日志 |
| `NODE_IP` | `string` | ❌ | `"localhost"` | - | 节点 IP 地址 |
| `NODE_LABELS` | `string` | ❌ | - | - | 节点标签（逗号分隔），用于节点池匹配 |

> **Nomad Node Pool 配置**
> - `node_pool` 不是环境变量，而是 Nomad Client 配置项
> - 在 `/etc/nomad.d/client.hcl` 中设置：`client { node_pool = "default" }`
> - 用于隔离 Orchestrator（`default` pool）和 Template Manager（`build` pool）
> - **关键**：两者默认端口相同（5008），必须通过 Node Pool 分离部署
| `DOMAIN_NAME` | `string` | ❌ | `""` | - | 服务域名 |

#### 服务控制

| 变量 | 类型 | 必需 | 默认值 | 枚举值 | 说明 |
|------|------|------|--------|--------|------|
| `ORCHESTRATOR_SERVICES` | `string` | ❌ | `"orchestrator"` | `orchestrator` `template-manager` | 运行的服务类型（逗号分隔） |
| `FORCE_STOP` | `bool` | ❌ | `false` | `true` `false` | 强制停止，跳过锁检查和 drain 等待 |
| `DISABLE_STARTUP_RECLAIM` | `bool` | ❌ | `false` | `true` `false` | 禁用启动时回收遗留资源 |
| `ORCHESTRATOR_LOCK_PATH` | `string` | ❌ | `/orchestrator.lock` | - | 进程锁文件路径 |

> **Orchestrator vs Template Manager 端口冲突**
> - 两者默认 `GRPC_PORT=5008`（上游为 Edge API 统一路由设计）
> - Nomad 部署必须通过 **Node Pool 分离**：
>   - Orchestrator → `node_pool = "default"` (system job)
>   - Template Manager → `node_pool = "build"` (service job)
> - API 对 Template Manager 的 Nomad/Kubernetes discovery 均使用 `ORCHESTRATOR_PORT`（默认 `5008`），不会读取自定义 allocation 端口。因此不能只把 Template Manager 改为 `5009`；同机共存时必须为两者提供不同的可达 IP/网络命名空间，并仍在各自 IP 上监听 `5008`

## 客户端环境变量

> ⚠️ **重要区分**：以下环境变量由 **E2B SDK/CLI 使用**，不属于服务端配置。它们用于客户端连接到 E2B API，而非服务启动参数。
>
> 适用范围：`e2b` Python SDK、`e2b-code-interpreter` Python SDK、E2B CLI 工具。

### SDK/CLI 认证变量

| 变量 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `E2B_API_KEY` | `string` | ✅（或 `E2B_ACCESS_TOKEN`） | - | E2B API 密钥，用于 SDK/CLI 认证。格式：`e2b_xxx`。从 Dashboard Team 标签页获取 |
| `E2B_ACCESS_TOKEN` | `string` | ✅（或 `E2B_API_KEY`） | - | E2B 个人访问令牌。从 Dashboard Personal 标签页获取 |
| `E2B_DOMAIN` | `string` | ❌ | `e2b.app` | E2B API 服务器域名。公共云使用默认值，私有化部署设置为自己的域名 |

### SDK/CLI 连接配置

| 变量 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `E2B_API_URL` | `string` | ❌ | `https://api.<E2B_DOMAIN>` | 完整 API URL。通常无需设置，调试时可指定 |
| `E2B_SANDBOX_URL` | `string` | ❌ | 自动构造 | Sandbox 连接 URL。详见下方说明 |
| `E2B_DEBUG` | `string` | ❌ | `false` | 启用调试模式。设为 `true` 时连接 `localhost:3000` |

#### E2B_SANDBOX_URL 详细说明

**URL 自动构造逻辑：**

| 条件 | URL 格式 | 示例 |
|------|----------|------|
| `E2B_SANDBOX_URL` 已设置 | 直接使用该值 | `https://proxy.example.com:8443` |
| `E2B_DEBUG=true` | `http://localhost:{port}` | `http://localhost:49983` |
| 默认（生产） | `https://{port}-{sandbox_id}.{domain}` | `https://3000-abc123.your-domain.com` |

**请求寻址约定（不是服务启动变量）：**

- 普通 Sandbox URL 通过 Host 的 `{port}-{sandboxID}.{E2B_DOMAIN}` 寻址。
- 共享入口使用 `sandbox.<E2B_DOMAIN>`，并同时发送 `E2b-Sandbox-Id` 与 `E2b-Sandbox-Port` Header。
- `localhost` 或 IP Host 也使用这两个 Header；IP Host 无法从 Host 编码 Sandbox ID 和端口。
- 普通 Sandbox Host 优先于 Header；在共享入口、localhost 或 IP 上缺少任一 Header 会返回 `400`。

```bash
curl https://sandbox.e2b.example.com/ \
  -H 'E2b-Sandbox-Id: isv6ril5xadwn1k9t2jye' \
  -H 'E2b-Sandbox-Port: 8080'
```

**使用场景：**

| 场景 | 是否需要设置 | 说明 |
|------|-------------|------|
| 标准私有化部署 | ❌ 不需要 | 设置 `E2B_DOMAIN` 即可，URL 自动构造 |
| 本地开发测试 | ❌ 不需要 | 设置 `E2B_DEBUG=true` |
| 非标准端口 | ✅ 需要 | 如果 client-proxy 不在 443/80 端口 |
| 自定义代理/CDN | ✅ 需要 | 使用自建代理或 CDN 时 |

**详细配置参考：** 参见 [`不修改代码完整部署指南.md#5-dns-配置`](../deployment/不修改代码完整部署指南.md#5-dns-配置)

### SDK/CLI HTTP 连接池配置

| 变量 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| `E2B_MAX_KEEPALIVE_CONNECTIONS` | `int` | ❌ | `20` | HTTP keepalive 连接池大小 |
| `E2B_MAX_CONNECTIONS` | `int` | ❌ | `2000` | HTTP 最大连接数 |
| `E2B_KEEPALIVE_EXPIRY` | `int` | ❌ | `300` | Keepalive 过期时间（秒） |

### 使用场景

| 场景 | 设置示例 |
|------|----------|
| 公共 E2B 云服务 | 仅设置 `E2B_API_KEY`，无需设置其他变量 |
| 私有化部署 | `E2B_API_KEY` + `E2B_DOMAIN=your-domain.com` |
| 本地开发测试 | `E2B_API_KEY` + `E2B_DEBUG=true` |
| 高并发场景 | 调整 `E2B_MAX_CONNECTIONS` 和 `E2B_MAX_KEEPALIVE_CONNECTIONS` |

### 常见误区

| 误区 | 正确理解 |
|------|----------|
| "E2B_API_KEY 是服务端配置" | ❌ 错误。它是客户端变量，用户在调用 API 时使用 |
| "E2B_DOMAIN 要在服务启动时设置" | ❌ 错误。它是 SDK/CLI 连接目标，服务端使用 `DOMAIN_NAME` |
| "私有化部署需要修改代码" | ❌ 错误。客户端设置 `E2B_DOMAIN` 即可，无需改代码 |
| "e2b-code-interpreter 有额外变量" | ❌ 错误。它完全继承 `e2b` 基础包的环境变量 |

### 相关代码位置

固定的 `infra` 子模块不包含 Python/JavaScript SDK 源码，因此不能用本仓库路径验证 SDK 版本的完整变量清单。上表仅用于区分服务端与客户端配置；实际变量、默认值和弃用状态应以所使用的 SDK/CLI 正式版本为准。`infra/packages/orchestrator/cmd/inspect-build/main.go` 只提供服务端仓库内工具使用这些变量的示例。
